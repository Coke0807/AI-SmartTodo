# Project Context / 项目背景
- 项目名称：SmartTodo AI (智能任务管家)

- 核心 AI 逻辑：
  - 双模驱动：系统必须支持 “本地轻量模型 (Ollama)” 和 “云端大模型 (OpenAI/DeepSeek API)” 两种调用方式。
  - 运行模式：支持单独运行本地模型、单独运行远程 API、或二者均启用根据网络状态自动降级切换。
    前端界面要包含具体的 apikey 配置和 baseurl 配置等常见的必须的 ai 模型配置面板。
  
  项目功能概述：一款结合 AI 大模型的智能待办事项 web 端应用，除去最基本的待办事项增删改查功能外，还需要结合 ai 大模型的能力来提高使用体验和基本的 ai 对话交互功能。
  
- 项目功能详情 / Functional Requirements：
  请基于以下功能列表进行排期规划，确保覆盖所有功能点：
  1. 用户与权限模块：
     - 用户注册/登录（JWT 无状态鉴权）。
     - 个人资料修改与密码重置。
  2. 智能任务管理 (Core)：
     - 任务 CRUD：创建、删除、修改、标记完成。
     - AI 智能拆解：输入“准备毕业答辩”，AI 自动拆解为“写 PPT”、“改论文”、“模拟演练”等子任务。
     - AI 预估耗时：根据任务描述，AI 预测完成所需时间。
     - 智能优先级：AI 根据截止日期和内容紧急程度，自动打标（P0/P1/P2）。
  3. AI 交互中心：
     - RAG 知识库问答：上传用户笔记（TXT/MD），AI 基于笔记内容回答问题（本地模型处理）。*注意：为精简架构，此处 RAG 优先采用内存向量或轻量级文件检索，不引入重型向量数据库。*
     - 全局 AI 助手：悬浮球 ChatBot，随时提问（云端模型处理）。
  4. 数据统计看板：
     - 本周完成任务数环形图、任务拖延率统计。

- 技术栈要求：
  - 前端：React 18 + Ant Design + TypeScript + Vite
  - 后端业务：Go (Gin) + GORM + PostgreSQL
  - AI 服务微服务：Python FastAPI (封装 langchain 或 openai SDK，实现 /v1/chat 接口，通过环境变量或配置切换)
  - 内部通信：Go 业务端通过 gRPC 调用 Python AI 微服务（需定义统一的 .proto 文件）
  - 部署：开发时本地运行，最终交付时通过 Docker & Docker Compose 将前后端、AI 服务、数据库四部分以单独容器解耦编排。
  
- 交付物：可一键启动的容器化系统 + 项目文档。
