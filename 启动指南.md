## SmartTodo 项目启动方法

项目采用三层微服务架构，需要按顺序启动 **4 个服务**：

---

### 1. 启动数据库 (PostgreSQL)

```bash
# 在项目根目录执行
docker compose up -d postgres
```

这是唯一依赖 Docker 的服务，启动后监听默认的 5432 端口。

---

### 2. 配置并启动后端 (Go)

**首次配置**：
```bash
cd backend
cp .env.example .env.local
# 编辑 .env.local，修改 DB_PASSWORD 等数据库连接信息
```

**启动命令**：
```bash
cd backend
go run ./cmd/server/
```

启动后监听 **:8080** 端口，提供 REST API 和 gRPC 客户端转发。

---

### 3. 启动 AI 服务 (Python)

```bash
cd ai_service
uv sync          # 安装/同步 Python 依赖
python main.py   # 启动服务
```

同时监听两个端口：
- **:50051** — gRPC 服务（供 Go 后端调用）
- **:8000** — FastAPI REST 接口（健康检查等）

---

### 4. 启动前端 (React)

```bash
cd frontend
pnpm install     # 安装依赖（首次）
pnpm dev         # 启动 Vite 开发服务器
```

启动后监听 **:5173** 端口，Vite 已配置将 `/api` 代理到后端 `:8080`。

---

### 启动顺序总结

```
Docker PostgreSQL  →  Go Backend(:8080)  →  Python AI Service(:50051/:8000)  →  React Frontend(:5173)
```

最后访问 **http://localhost:5173** 即可使用。

---

### 前提条件

你的机器需要已安装：
- **Docker** — 运行 PostgreSQL
- **Go** — 编译运行后端
- **Python 3.12+** + **uv** — 运行 AI 服务
- **Node.js** + **pnpm** — 运行前端
